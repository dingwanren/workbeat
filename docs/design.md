2. 团队成员分析页 (Team Velocity & Rhythm)
这是核心页面，用于分析“人”的数据。

成员列表/卡片矩阵：
左侧是一个成员列表，点击某人，右侧展示详情；或者直接平铺卡片。

核心图表设计：
工作时段分布 (Radar Chart / Bar Chart):
X轴：0点到24点。
Y轴：提交频率。
用途：一眼看出谁在熬夜，谁在朝九晚五。

代码产出趋势 (Stacked Area Chart - 堆叠面积图):
X轴：时间。
Y轴：代码行数。
图例：新增(绿色)、删除(红色)。
用途：观察某个人的输出是否稳定，还是突击式提交。
提交习惯散点图 (Scatter Plot):
X轴：提交包含的文件数量 (Impact Scope)。
Y轴：提交包含的代码行数 (Size)。
每个点代表一次 Commit。
用途：识别“原子提交”习惯。点都在左下角说明习惯好（多次少量提交）；点在右上角说明喜欢憋大招（一次提交巨大改动）。



3. 代码质量与趋势页 (Code Insights)
关注代码本身的变化，而非人的绩效。
文件热点图 (Treemap - 矩形树图):
界面展示一个个矩形，矩形大小代表文件大小，颜色深浅代表修改次数。
视觉效果： 一眼看到红色的矩形，那就是项目中改动最频繁的文件（通常也是Bug高发区）。
语言构成 (Language Breakdown):
展示 TypeScript, Rust, Python 等语言的行数占比及变化趋势。



交互体验 (UX)
Drill Down (下钻): 点击 Dashboard 上的某个仓库柱状图，自动跳转到该仓库的详情页。
Tooltips: 鼠标悬停在图表上时，要显示详细信息（例如：Hash值、Commit Message摘要）。
Loading 状态: 因为分析多个仓库可能需要几秒钟，界面初始化时搞一个终端风格的 Loading 动画（比如绿色的字符雨或者进度条）。
